# Competitive Research — 2026-03-13

## Chong-U (@chongdashu) — Sprite Generation Technique

**Post**: https://x.com/chongdashu/status/2032041412590551293 (12 Mar 2026, 14.7K views)
**Article**: "Generating Animated Game Sprites using GPT 5.4 + Image 1.5"

### Technique:
1. **Static spritesheets**: Use GPT 5.4 + Image 1.5 to generate pixel art sprite sheets directly from text prompts
2. **Walk cycles** (the hard part): Direct image generation can't nail walk cycles
   - **Solution**: Generate video with **Sora 2** → Extract frames → Stitch into spritesheet
   - "The results are actually quite promising"

### Applicability to Kurokimachi:
- Our building sprites are generated by Mosaic via Gemini image generation
- We could upgrade to GPT 5.4 Image 1.5 or similar for higher quality static sprites
- For animated agent walk cycles: use video-to-spritesheet pipeline (Sora 2 or equivalent)
- Could create animated building effects (smoke, lights, activity) same way

## Luke The Dev (@iamlukethedev) — Star-Office-UI / Pixel Agents

**Profile**: https://x.com/iamlukethedev (3.2K followers)
**Video**: "I Can Actually Watch My AI Agents Work Now" — https://youtube.com/watch?v=62Rfe1w9NBc
**Repo**: https://github.com/ringhyacinth/Star-Office-UI

### What he has:
- **Pixel art office dashboard** for AI agents
- **6 agent states** (idle/writing/researching/executing/syncing/error) → agents move to corresponding areas
- **Voice control**: Press mic, tell agent what to do — "feels like managing employees"
- **Multi-agent collaboration**: Join keys, guest agents, real-time status
- **"Yesterday memo"**: Auto-reads memory/*.md, displays as card
- **AI-generated decoration**: Gemini API for office backgrounds
- **Desktop pet mode**: Electron transparent window
- **Mobile responsive**: Quick glance on phone
- **State sync**: Agents auto-push status via scripts

### What kurokimachi has that he doesn't:
- Full isometric town world (not just an office)
- Building/upgrade system with levels
- Citizen roster with distinct personalities
- Gazette/news feed of world events
- Lore (sacred structures, districts, history)
- Autonomous world building cycle (agents evolve the town)

### What he has that we should build:
1. **Agent state visualization** — show what each agent is actively doing (idle at home, working at workshop, researching at library). We already have buildings themed for this — just need state→location mapping.
2. **Voice control** — talk to Scarlet or any agent via mic button. We have the text input box already.
3. **Yesterday memo / activity summary** — visible card showing recent agent activity beyond just gazette
4. **Better sprite quality** — his pixel art is more polished. Use Chong-U's technique.
5. **Real-time state indicators** — status bubbles, activity animations on buildings when agent is working there

### Key difference in philosophy:
- Luke's is an **office** — agents sit at desks and work. It's a workspace visualization.
- Ours is a **world** — agents are citizens living in a town. It's a living simulation.
- Our approach is more ambitious and distinctive. Don't copy the office vibe — enhance the town/world vibe.
